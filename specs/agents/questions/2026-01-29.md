## Phase 2: Modular Agent Architecture

### Q2.1: Router vs Generalist (Critical Decision)
**Option A: Router Model**
- Small model (e.g., gpt-4o-mini) selects appropriate agent
- Similar to Legacy Skyth's Router class
- Explicit routing logic

**Option B: Generalist Agent with Subagent Calling**
- Single generalist agent with `task` tool (like Skyth)
- Generalist decides when to delegate to specialized agents
- More autonomous, fewer components

**Question:** Which approach should we implement? 

**My Recommendation:** Option B (Generalist with subagent calling) because:
- Simpler architecture (fewer components)
- More flexible (generalist can adapt routing logic)
- Follows Skyth's proven pattern
- Lower latency (no separate routing LLM call)

**If you choose Option A (Router):**
- Which model for routing? (gpt-4o-mini? gemini-flash?)
- Should router have fallback logic if selected agent unavailable?

**If you choose Option B (Generalist):**
- How should agents call subagents? (Tool-based like Skyth's `task` tool?)
- What context do subagents receive? (Full history or task-specific?)

### Q2.3: MCP to TOON Converter
You mentioned "converter that transforms MCP servers to use TOON."

**Understanding from TOON README:**
- TOON is a token-efficient JSON encoding format
- Designed for LLM input (not output generation)
- ~40% fewer tokens than JSON for tabular data
- CSV-like for uniform arrays, YAML-like for nested objects

**Questions:**
1. **Converter purpose:** Should we:
   - Convert MCP tool schemas (JSON) to TOON before sending to LLM?
   - Convert MCP tool results (JSON output) to TOON before passing to LLM?
   - Both?
2. **When to use TOON vs JSON:**
   - Always convert everything to TOON?
   - Only convert large data structures (>N tokens)?
   - Let user configure per-tool?
3. **Implementation approach:**
   - Use existing TOON library (Python: `toon_format`)?
   - Write custom converter optimized for MCP schemas?
4. **Error handling:** If TOON conversion fails (e.g., deeply nested non-uniform data), fallback to JSON?

**Example Use Case:**
```python
# MCP server returns large dataset
mcp_result = {
  "users": [
    {"id": 1, "name": "Alice", "role": "admin"},
    {"id": 2, "name": "Bob", "role": "user"},
    # ... 100 more users
  ]
}

# Converter transforms to TOON
toon_result = """
users[102]{id,name,role}:
  1,Alice,admin
  2,Bob,user
  ...
"""

# Send TOON to LLM (saves ~40% tokens)
```

**Is this the intended flow?**
